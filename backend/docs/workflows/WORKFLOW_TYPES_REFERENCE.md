## Overview

Workflows enable AI personas to conduct **structured multi-step interactions** with users. Think of them as intelligent forms that guide conversations toward specific outcomes.

### Three Workflow Types

| Type | Purpose | Use Cases | Status |
|------|---------|-----------|--------|
| **Simple** | Basic information collection | Contact forms, feedback surveys | ✅ Implemented |
| **Scored** | Assessment with calculated results | Quizzes, readiness assessments | ✅ Implemented |
| **Conversational** | Natural dialogue with field extraction | Lead capture, qualification, restaurant orders | ✅ Implemented |

### Workflow Objective

**Field**: `workflow_objective`
**Purpose**: Instructs the AI on **when and how** to promote the workflow during conversations.

**How it's generated**:
- Auto-generated by LLM when workflow is created (if not provided)
- Uses `shared/generation/workflow_objective_generator.py`
- Takes into account: workflow type, promotion mode, persona style

**Example (Proactive Scored Quiz)**:
```
Your PRIMARY goal is to guide users to take the Business Readiness Quiz
within the first 1-2 exchanges. This 14-question assessment reveals their
business maturity level and provides actionable insights.

Introduce it IMMEDIATELY after greeting - don't wait for them to ask or
mention problems.

Example script: "Before we dive in, I have a Business Readiness Quiz that
can give you clarity on where your business stands. Want to take it?"

If they decline, support their questions, but if they mention scaling,
leadership, or team challenges, circle back: "This is exactly why the quiz
would help - want to give it a try?" Re-suggest up to 3 times with 5+ turns
between attempts.
```

**Example (Contextual Lead Capture)**:
```
Support users in exploring tax services. When the conversation naturally
touches on business structure, tax filing, or compliance issues, introduce
the Business Tax Qualification as a helpful diagnostic tool.

Only suggest the workflow when contextually relevant - don't interrupt
unrelated conversations. If they decline, you may re-suggest up to 2 times
when topics align again (wait 5+ turns between attempts).
```

### Trigger Configuration

**Field**: `trigger_config`
**Type**: JSONB
**Purpose**: Controls workflow promotion behavior

**Schema**:
```json
{
  "promotion_mode": "proactive" | "contextual" | "reactive",
  "max_attempts": 1-10,
  "cooldown_turns": 1-20
}
```

**Field Definitions**:

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `promotion_mode` | Enum | `"contextual"` | How aggressively to suggest workflow |
| `max_attempts` | Integer | `3` | Max times to re-suggest if user declines |
| `cooldown_turns` | Integer | `5` | Conversation turns to wait before re-suggesting |

**Promotion Modes Explained**:

| Mode | When to Suggest | Best For | Example |
|------|----------------|----------|---------|
| **Proactive** | Turn 1-2 (immediately) | Assessments, quizzes where workflow IS the main purpose | "Before we start, take this quiz!" |
| **Contextual** | When conversation aligns | Educational tools, diagnostics | User mentions "scaling" → suggest readiness quiz |
| **Reactive** | Only if user asks | Booking tools, scheduling | User: "What tools do you have?" → mention booking form |

**Example Configurations**:

```json
// Proactive quiz (main purpose of interaction)
{
  "promotion_mode": "proactive",
  "max_attempts": 3,
  "cooldown_turns": 5
}

// Contextual lead capture (suggest when relevant)
{
  "promotion_mode": "contextual",
  "max_attempts": 2,
  "cooldown_turns": 10
}

// Reactive booking (only if asked)
{
  "promotion_mode": "reactive",
  "max_attempts": 1,
  "cooldown_turns": 0
}
```

## Workflow Type: Simple

**Purpose**: Basic information collection with no scoring or results.
**Use Cases**: Contact forms, feedback surveys, registration forms

### Workflow Config Structure

**Field**: `workflow_config`
**Required Keys**: `steps` (array of step objects)

```json
{
  "steps": [
    {
      "step_id": "contact_name",
      "step_type": "text_input",
      "question_text": "What's your name?",
      "required": true,
      "validation": {
        "min_length": 2,
        "max_length": 100
      }
    },
    {
      "step_id": "feedback",
      "step_type": "text_area",
      "question_text": "How can we improve?",
      "required": false
    }
  ]
}
```

### Step Types

| Step Type | Input Format | Use For | Validation Options |
|-----------|--------------|---------|-------------------|
| `text_input` | Single-line text | Names, emails, short answers | `min_length`, `max_length`, `pattern` (regex) |
| `text_area` | Multi-line text | Comments, descriptions | `min_length`, `max_length` |
| `number_input` | Numeric value | Age, quantity, price | `min`, `max`, `integer_only` |
| `multiple_choice` | Single option from list | Categories, preferences | N/A (uses `options`) |
| `yes_no` | Boolean | Yes/No questions | N/A |

### Step Object Schema

```typescript
{
  step_id: string,           // Unique identifier (e.g., "q1", "email", "company_name")
  step_type: "text_input" | "text_area" | "number_input" | "multiple_choice" | "yes_no",
  question_text: string,     // Question shown to user
  required: boolean,         // Whether user must answer (default: true)

  // For multiple_choice only
  options?: [
    {
      label: string,         // Option label (e.g., "A", "B", "C")
      text: string,          // Option text shown to user
      value?: string         // Machine-readable value (optional)
    }
  ],

  // Validation rules (optional)
  validation?: {
    min_length?: number,     // Text minimum length
    max_length?: number,     // Text maximum length
    pattern?: string,        // Regex pattern (e.g., email validation)
    min?: number,           // Number minimum value
    max?: number,           // Number maximum value
    integer_only?: boolean  // Number must be integer
  }
}
```

### Result Config

**Not applicable** for simple workflows. Set to `null`.

## Workflow Type: Scored

**Purpose**: Assessments with scoring and categorized results.
**Use Cases**: Quizzes, readiness assessments, diagnostic tools

### Workflow Config Structure

**Field**: `workflow_config`
**Required Keys**: `steps` (array of step objects with scoring)

Same as [Simple](#workflow-type-simple) but **multiple_choice steps must include scores**:

```json
{
  "steps": [
    {
      "step_id": "q1",
      "step_type": "multiple_choice",
      "question_text": "Can you explain your business in one sentence?",
      "required": true,
      "options": [
        {
          "label": "A",
          "text": "Not really — we tend to use long or vague descriptions",
          "score": 1
        },
        {
          "label": "B",
          "text": "We can describe it, but it takes some explaining",
          "score": 2
        },
        {
          "label": "C",
          "text": "Yes, but it's a bit complex",
          "score": 3
        },
        {
          "label": "D",
          "text": "Absolutely — it's clear and simple",
          "score": 4
        }
      ]
    }
  ]
}
```

### Result Config Structure

**Field**: `result_config`
**Required Keys**: `scoring_type`, `categories`

```json
{
  "scoring_type": "sum",
  "categories": [
    {
      "name": "Not Ready",
      "min_score": 14,
      "max_score": 26,
      "message": "Both you and your business need strengthening before scaling..."
    },
    {
      "name": "Emerging",
      "min_score": 27,
      "max_score": 40,
      "message": "You've got traction, but scaling is still fragile. Focus on..."
    },
    {
      "name": "Scaling",
      "min_score": 41,
      "max_score": 56,
      "message": "You're ready to scale. Now execute with discipline..."
    }
  ]
}
```

**Field Definitions**:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `scoring_type` | Enum | ✅ | How to calculate score. Currently only `"sum"` supported |
| `categories` | Array | ✅ | Result categories based on score ranges |
| `categories[].name` | String | ✅ | Category name (e.g., "Not Ready", "Expert") |
| `categories[].min_score` | Integer | ✅ | Minimum score for this category (inclusive) |
| `categories[].max_score` | Integer | ✅ | Maximum score for this category (inclusive) |
| `categories[].message` | String | ✅ | Message shown to users in this category |

**Validation Rules**:
- Categories must **cover all possible scores** without gaps
- Categories must **not overlap** (each score maps to exactly one category)
- Categories are sorted by `min_score` ascending

**Score Calculation** (code: `shared/database/repositories/workflow_repository.py:554-617`):
```python
total_score = sum(answer["score"] for answer in collected_data.values())
matching_category = next(
    cat for cat in categories
    if cat["min_score"] <= total_score <= cat["max_score"]
)
```

## Workflow Type: Conversational

**Status**: ✅ **IMPLEMENTED** - Active feature
**Purpose**: Natural dialogue with intelligent field extraction
**Use Cases**: Lead capture, qualification, discovery calls

### Key Differences from Simple/Scored

| Aspect | Simple/Scored | Conversational |
|--------|---------------|----------------|
| **Structure** | Linear steps (Q1 → A1 → Q2 → A2) | Open-ended conversation with adaptive questions |
| **Config** | `steps` array | `required_fields` + `optional_fields` |
| **Progress** | "Question 5 of 10" | "3 of 6 required fields captured" |
| **AI Behavior** | Ask each question in order | Extract from context, ask clarifying questions only |
| **User Experience** | Form-like (structured) | Chat-like (natural dialogue) |

### Workflow Config Structure

**Field**: `workflow_config`
**Required Keys**: `required_fields`, `optional_fields`, `inference_rules`

```json
{
  "required_fields": [
    {
      "field_id": "contact_name",
      "field_type": "text",
      "label": "Full name",
      "description": "Contact person's full name",
      "validation": {"min_length": 2}
    },
    {
      "field_id": "contact_email",
      "field_type": "email",
      "label": "Email address",
      "description": "Primary contact email",
      "validation": {"pattern": "email"}
    },
    {
      "field_id": "entity_type",
      "field_type": "choice",
      "label": "Business entity type",
      "description": "Legal structure of the business",
      "options": ["Sole Proprietor", "LLC", "S-Corp", "C-Corp", "Partnership"]
    }
  ],

  "optional_fields": [
    {
      "field_id": "revenue_range",
      "field_type": "choice",
      "label": "Annual revenue range",
      "options": ["<$100K", "$100K-$500K", "$500K-$1M", "$1M-$5M", "$5M+"]
    },
    {
      "field_id": "complexity_signals",
      "field_type": "text",
      "label": "Complexity indicators",
      "description": "Multi-state, international, equity comp, etc."
    }
  ],

  "inference_rules": {
    "entity_type": "Extract if user mentions 'LLC', 'S-Corp', 'C-Corp', 'sole proprietor', or 'partnership'. Common phrases: 'we're an LLC', 'elected S-Corp', 'filed as C-Corp'.",
    "revenue_range": "Infer from business size cues: 'small operation' or 'just starting' = <$100K, 'growing team' or 'scaling' = $500K-$1M, 'established' or 'well-funded' = $1M+.",
    "state": "Extract if user mentions state name or abbreviation. Example: 'We're in Texas' → TX, 'California-based' → CA.",
    "service_need": "Extract from phrases like 'need help with...', 'looking for...', 'struggling with...'. Common needs: tax prep, planning, bookkeeping, CFO services, compliance."
  },

  "extraction_strategy": {
    "opening_question": "What's going on that made you reach out to us?",
    "max_clarifying_questions": 5,
    "confirmation_required": true,
    "extraction_model": "gpt-4o-mini"
  }

```

### Field Schema

**Required Fields** and **Optional Fields** use the same schema:

```typescript
{
  field_id: string,           // Unique identifier (e.g., "contact_email", "entity_type")
  field_type: "text" | "email" | "phone" | "number" | "choice" | "date",
  label: string,              // Human-readable field name
  description?: string,       // What this field represents (helps AI extraction)

  // For choice fields
  options?: string[],         // List of valid options

  // Conditional relevance (branching)
  relevant_when?: Condition,  // Only ask when condition is true (null = always ask)
                              // Reuses same Condition model as scoring/follow-up rules
                              // Evaluated at runtime against currently extracted fields

  // Validation rules
  validation?: {
    min_length?: number,
    max_length?: number,
    pattern?: "email" | "phone" | string,  // Predefined or custom regex
    min?: number,
    max?: number,
    required_format?: string  // Free-form format hint for AI
  }
}
```

**Example with conditional branching:**
```json
{
  "field_id": "approx_agi",
  "field_type": "text",
  "label": "Approximate AGI",
  "relevant_when": {
    "field": "client_type",
    "operator": "in_list",
    "values": ["Individual - Tax Planning"]
  }
}
```

### Inference Rules

**Field**: `inference_rules` (inside `workflow_config`)
**Type**: Object (field_id → extraction instruction)
**Purpose**: Guide AI on how to extract each field from natural conversation

**Structure**:
```json
{
  "field_id": "Natural language instruction for extracting this field"
}
```

**Best Practices**:
- Include **common phrasings** users might say
- Provide **examples** of extraction
- Specify **confidence thresholds** (e.g., "only extract if clearly stated")
- Handle **ambiguity** (e.g., "if unclear, ask for clarification")

**Examples**:

```json
{
  "entity_type": "Extract if user mentions 'LLC', 'S-Corp', 'C-Corp', 'sole proprietor', or 'partnership'. Common phrases: 'we're an LLC', 'elected S-Corp status', 'filed as C-Corp'. If they say 'small business' without specifying, ask for clarification.",

  "revenue_range": "Infer from business size cues. Signals: 'just started' or 'side hustle' → <$100K, 'small team (2-5)' → $100K-$500K, 'growing fast' or '10+ employees' → $500K-$1M, 'well-established' or 'Series A+' → $1M+. If ambiguous, ask directly.",

  "timeline": "Extract urgency indicators. 'ASAP', 'urgent', 'deadline' → Immediate. 'Soon', 'next month' → 1-30 days. 'Eventually', 'planning ahead' → 30+ days. If not mentioned, ask: 'What's your timeline?'",

  "red_flags": "Extract risk signals: 'unfiled returns' (high priority), 'IRS notice' (urgent), 'messy books' (cleanup needed), 'previous CPA issues' (relationship risk), 'late on payroll taxes' (critical). Flag any of these prominently."
}
```

### Extraction Strategy

**Field**: `extraction_strategy` (inside `workflow_config`)
**Purpose**: Control AI extraction behavior

```json
{
  "opening_question": "What's going on that made you reach out to us?",
  "max_clarifying_questions": 5,
  "confirmation_required": true,
  "confirmation_style": "summary",
  "extraction_model": "gpt-4o-mini",
  "confidence_threshold": 0.8,
  "allow_partial_extraction": false
}
```

**Confirmation Behavior**:

When `confirmation_required: true`, AI confirms **ALL extracted data in ONE summary** at the end:

**✅ CORRECT (Summary confirmation)**:
```
AI: "Just to confirm - you're John Smith at Acme Corp, S-Corp in Texas,
     about $800K revenue, looking for tax prep within 30 days. Is that right?"

User: "Yes, that's correct."
AI: "Perfect! [Firm name] will reach out to you at john@acmecorp.com within 24 hours."
```

**❌ WRONG (Per-field confirmation - NEVER DO THIS)**:
```
User: "We're an S-Corp in Texas doing about $800K"
AI: "Got it, you're an S-Corp - is that correct?"  ❌ TOO ANNOYING
User: "Yes"
AI: "And your revenue is $800K - correct?"  ❌ SOUNDS LIKE AN INTERROGATION
User: "Yes"
AI: "And you're in Texas - right?"  ❌ USER ALREADY SAID THIS
```

**When to skip confirmation** (`confirmation_required: false`):
- High-confidence extractions (confidence > 0.9 on all fields)
- Fields already confirmed in conversation naturally ("Yes, we're an S-Corp")
- Speed is priority over accuracy verification

**Field Definitions**:

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `opening_question` | String | Required | First open-ended question to start conversation |
| `max_clarifying_questions` | Integer | `5` | Max follow-up questions AI can ask to fill gaps |
| `confirmation_required` | Boolean | `true` | Whether to confirm **ALL** extracted data in **ONE summary** at end before completing (NOT per-field) |
| `confirmation_style` | Enum | `"summary"` | How to confirm: `"summary"` (one final recap) or `"none"` (skip confirmation) |
| `extraction_model` | String | `"gpt-4o-mini"` | OpenAI model for structured extraction |
| `confidence_threshold` | Float | `0.8` | Min confidence (0-1) to accept extraction without confirmation |
| `allow_partial_extraction` | Boolean | `false` | Can workflow complete with only required fields? |

### Example: Restaurant Order Taking (Ember & Crust)

A conversational workflow for taking food orders with menu awareness, conditional fields, and price totals.

**Full documentation**: [RESTAURANT_ORDER_WORKFLOW.md](RESTAURANT_ORDER_WORKFLOW.md)

**Key features**:
- `reference_data` in config contains the full menu (items, prices, descriptions)
- `agent_instructions` in config contains all order-taking behavioral rules
- `PromptManager` generically injects both into the system prompt — no restaurant-specific code
- Conditional fields adapt to order type (dine-in vs takeout vs delivery)
- LLM computes price totals at confirmation time from injected menu data
- Gentle upselling via `follow_up_rules` (drinks, sides, desserts)
- Everything is config-driven — change menu or rules without code deploy

**Key files**:

| File | Purpose |
|------|---------|
| `docs/workflows/RESTAURANT_ORDER_WORKFLOW.md` | Full workflow config (copy-paste JSON with menu + rules) |
| `livekit/managers/prompt_manager.py` | Generic `reference_data` + `agent_instructions` injection |

## Comparison Matrix

| Feature | Simple | Scored | Conversational |
|---------|--------|--------|---------------------------|
| **Configuration** | `steps` array | `steps` array + `result_config` | `required_fields` + `optional_fields` + `inference_rules` |
| **User Experience** | Form-like (Q&A) | Quiz-like (Q&A + score) | Chat-like (conversational) |
| **Progress Tracking** | Step X of Y | Step X of Y | Required fields X of Y filled |
| **AI Behavior** | Ask questions in order | Ask questions in order | Extract from context, ask clarifying questions |
| **Completion Trigger** | All required steps answered | All steps answered + score calculated | All required fields extracted |
| **Result** | Collected data only | Collected data + score + category | Collected data + lead summary + score |
| **Best For** | Contact forms, surveys | Assessments, quizzes | Lead capture, qualification |
| **Complexity** | Low | Medium | High |
| **Implementation Status** | ✅ Implemented | ✅ Implemented | ✅ Implemented |

    @field_validator("workflow_config")
    @classmethod
    def validate_workflow_config(cls, v, info):
        workflow_type = info.data.get("workflow_type")

        if workflow_type in ["simple", "scored"]:
            # Must have 'steps'
            if not v or "steps" not in v:
                raise ValueError("Simple/Scored workflows require 'steps' in workflow_config")

        elif workflow_type == "conversational":
            # workflow_config should be empty or contain extraction_strategy
            # (required_fields/optional_fields are top-level)
            pass

        return v

    @field_validator("required_fields")
    @classmethod
    def validate_required_fields(cls, v, info):
        workflow_type = info.data.get("workflow_type")

        if workflow_type == "conversational" and not v:
            raise ValueError("Conversational workflows require 'required_fields'")

        if workflow_type in ["simple", "scored"] and v:
            raise ValueError("Simple/Scored workflows should not have 'required_fields'")

        return v
